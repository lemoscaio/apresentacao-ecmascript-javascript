Onde entra o ECMA do ECMAScript?

ECMA -> European Computer Manufacturers Association (Associação Europeia de Fabricantes de Computadores).

Em 96, a Netscape mandou o Javascript para a associação que padronizava sistemas de informação, a ECMA. E já que o nome Javascript já era patenteado, a ECMA criou o ECMAscript, a nova versão padronizada da linguagem.

Inclusive, é possível consultar o padrão atual do ECMAScript através do link de publicação do ECMA-262.

Também é possível consultar o repisitório oficial do ECMA 262 (número de registro do ECMAScript) e verificar, por exemplo, propostas de padrões a serem adicionados em próximas versões.

Então o JavaScript que conhecemos é, na verdade, o ECMAscript?

Exatamente, o nome Javascript continua no senso comum pela sua popularidade.

Confusão dos nomes:

O ECMAScript era nomeado seguindo o padrão mais simples possível, ECMAScript 2, ECMAScript 3...

A partir do ECMAScript 6 (o famoso ES6, onde muita coisa nova chegou), mudaram o padrão de nomenclatura para acompanhar o ano de lançamento.
EX: ECMASCript 2015 / ES2015

No fim, o que importa é saber que, quando nos referimos a versão do Javascript, utilizamos o nome ECMAScript, mas que dificilmente o nome Javascript será retirado da senso comum
